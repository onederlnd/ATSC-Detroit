# Simulation — Michigan Ave & Livernois

This directory contains all SUMO input files for the ATSC-Detroit simulation.

---

## Network Geometry

The network was generated using SUMO's `netgenerate` tool as a 2x2 grid:

```bash
netgenerate --grid --grid.number=2 --tls.set=A0,A1,B0,B1 --output-file simulation/intersection.net.xml
```

### Junction Layout

```bash
A1 ——— B1
|       |
A0 ——— B0
```

Four junctions, each with a traffic light. The simulation controls junction **A0**, which represents the Michigan Ave & Livernois intersection. A0, A1, B0, B1 correspond to the four corners of the grid.

### Edge Naming Convention

Edges are named by their source and destination junction:

| Edge | Direction | Real-world analog |
| --- | --- | --- |
| `A1A0` | Southbound into A0 | Livernois Ave southbound |
| `B0A0` | Westbound into A0 | Michigan Ave westbound approach |
| `A0B0` | Eastbound out of A0 | Michigan Ave eastbound departure |
| `A0A1` | Northbound out of A0 | Livernois Ave northbound departure |

### Limitations

The current network is a simplified proxy. It does not reflect the actual lane geometry, turning movements, or signal phasing of the real Michigan Ave & Livernois intersection. It serves as a functional simulation environment for RL development. See the [Roadmap](../README.md#roadmap) for plans to replace it with real geometry.

## Traffic Signal

**Controlled junction:** `A0`  
**Traffic light ID:** `A0`  
**Number of phases:** 4  
**Phase structure:** Defined by `netgenerate` based on junction geometry.

The RL agent selects which phase to activate at each timestep. Yellow transitions (4 seconds) are enforced automatically by `traffic_env.py` — the agent never directly controls yellow phases.

Signal constraints enforced by the environment:

| Constraint | Value | Reason |
| --- | --- | --- |
| Minimum green | 10 seconds | Prevents rapid flickering |
| Maximum green | 60 seconds | Prevents indefinite starvation |
| Yellow duration | 4 seconds | Standard urban arterial |

## Detector Placement

Inductive loop detectors are defined in `detectors.add.xml`. One detector per approach lane, placed 10 meters before the stop line.

| Detector ID | Lane | Approach |
| --- | --- | --- |
| `det_south_in` | `A0A1_0` | Northbound (vehicles entering from south) |
| `det_north_in` | `A1A0_0` | Southbound (vehicles entering from north) |
| `det_west_in` | `B0A0_0` | Westbound (vehicles entering from west) |
| `det_east_in` | `A0B0_0` | Eastbound (vehicles entering from east) |

Detectors report `getLastStepVehicleNumber` each simulation second. This value is used to build the observation vector in `traffic_env.py`.

**Note:** Production systems use multiple detectors per lane for queue length estimation. This simulation uses one detector per approach as a simplification.

## Demand Calibration

Vehicle demand is defined in `routes.rou.xml` and generated by `demand/generate_demand.py`.

### Source Data

| Road | AADT | Source |
| --- | --- | --- |
| Michigan Avenue | ~20,000 vpd | MDOT Traffic Volumes Open Data (2022) |
| Livernois Avenue | ~10,000 vpd | MDOT Traffic Volumes Open Data (2022) |

### Methodology

Peak hour volumes derived from AADT using K-factor = 0.09 (standard urban arterial). Off-peak volumes at 44% of peak.

| Period | Michigan Ave (total) | Livernois (total) |
| --- | --- | --- |
| AM Peak (7–9 AM) | 1,800 veh/hr | 900 veh/hr |
| Off-Peak (9AM–4PM) | 800 veh/hr | 400 veh/hr |
| PM Peak (4–6 PM) | 2,000 veh/hr | 1,000 veh/hr |

Directional splits: 55/45 WB/EB on Michigan Ave in AM (reversed PM), 50/50 on Livernois.

### Current Simulation Scope

`MAX_STEPS = 3600` (1 hour). Only AM peak flows fire in the current 1-hour simulation. To model a full day, increase `MAX_STEPS` to `86400` in `traffic_env.py`.

### Regenerating Demand

To update volumes and regenerate `routes.rou.xml`:

```bash
# Edit AADT values in simulation/demand/generate_demand.py
python simulation/demand/generate_demand.py
```

## Output Files

SUMO writes output to `simulation/output/` (created at runtime):

| File | Contents |
| --- | --- |
| `summary.xml` | Per-step aggregate stats (vehicles, waiting time, speed) |
| `tripinfo.xml` | Per-vehicle trip data (departure, arrival, waiting time) |
| `det_*.xml` | Per-detector counts (one file per inductive loop) |

These files are gitignored. They are regenerated each run.

## Running SUMO Directly

To run the simulation without the RL agent (useful for debugging demand):

```bash
# Headless
sumo -c simulation/intersection.sumocfg

# With GUI
sumo-gui -c simulation/intersection.sumocfg
```
